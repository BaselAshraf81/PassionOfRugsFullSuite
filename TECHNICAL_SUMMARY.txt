PROJECT OVERVIEW
================================================================================
A professional lead processing and dialer application designed for customer 
outreach campaigns. The system integrates with TrestleIQ reverse lookup APIs, 
CloudTalk VoIP calling platform, and OpenAI GPT for intelligent contact 
management. Built with Python/Tkinter for Windows desktop deployment.

Primary Use Case: Re-engaging past customers from 2015 purchase records by 
finding current contact information through reverse phone/address lookups.


SYSTEM ARCHITECTURE
================================================================================

Core Components:
1. Launcher (launcher.py) - Mode selection interface
2. Professional Dialer GUI (dialer_gui.py) - Interactive calling interface
3. Bulk Processor GUI (bulk_processor_gui.py) - Batch processing interface
4. Lead Processor (lead_processor_v2.py) - API integration & data processing
5. Cache Manager (cache_manager.py) - Persistent API response caching
6. AI Assistant (ai_assistant.py) - Address correction & contact filtering
7. Configuration (config.py) - API keys and settings

Total Codebase: ~6,251 lines across 7 Python files


OPERATING MODES
================================================================================

MODE 1: BULK PROCESSING
------------------------
Purpose: Batch process large Excel datasets with progress tracking
Features:
- Excel file input/output with formatted results
- Progress bar and status updates
- Dual caching system (Excel + permanent JSON cache)
- Color-coded results (RED = address lookup failed, GREEN = name match)
- Dropdown lists for multiple phones/addresses per person
- Status tracking column
- Automatic activity score filtering (threshold: 30)

Workflow:
1. Load Excel file with columns: name, phone, address, city, state, zip
2. Filter rows by activity_score <= 30
3. For each row: perform phone lookup + address lookup
4. Cache results to avoid duplicate API calls
5. Export formatted Excel with all found contacts
6. One row per person with all their phones/addresses


MODE 2: PROFESSIONAL DIALER
----------------------------
Purpose: Interactive calling interface for manual lead review
Features:
- Person-by-person navigation through Excel dataset
- One-click CloudTalk calling integration
- Manual/automatic API lookups (phone & address)
- Visual response viewer with HTML rendering
- Google Maps integration for addresses
- Call history tracking (one entry per phone number)
- AI-powered contact filtering and ranking
- Address correction with automatic retry
- Excel output with status tracking
- Settings panel for API control

Workflow:
1. Load Excel file and configure starting row
2. Navigate through persons (Previous/Next buttons)
3. View original data + API lookup results
4. AI analyzes and ranks contacts by likelihood
5. Click phone numbers to initiate CloudTalk calls
6. Mark status (Interested/Not Interested/No Answer/Wrong Number)
7. Add notes and save to Excel
8. Call history automatically tracks all calls


API INTEGRATIONS
================================================================================

1. TRESTLEIQ REVERSE LOOKUP API
--------------------------------
Base URL: https://api.trestleiq.com
Authentication: x-api-key header

Endpoints Used:
- GET /3.2/phone - Reverse phone lookup
  Parameters: phone (E.164 format: +1XXXXXXXXXX)
  Returns: owners[], alternate_phones[], current_addresses[]

- GET /3.1/location - Reverse address lookup
  Parameters: street_line_1, city, state_code, postal_code
  Returns: current_residents[], phones[], address validation

Rate Limiting: 0.1 second delay between calls
Cost Model: Pay-per-lookup (exact pricing not specified)

Data Extracted:
- Person names, ages, age ranges
- Phone numbers (mobile, landline, VoIP types)
- Current and historical addresses
- Carrier information
- Address validation status


2. CLOUDTALK CALLING API
-------------------------
Base URL: https://my.cloudtalk.io/api
Authentication: Basic Auth (base64 encoded access_key_id:access_key_secret)

Endpoint Used:
- POST /calls/create.json
  Parameters: agent_id, callee_number
  Returns: success status

Response Codes:
- 200: Call initiated successfully
- 403: Agent not online
- 406: Invalid phone number format
- 409: Agent already on call

Requirements:
- Agent must be logged into CloudTalk desktop/web app
- Agent must be in "Available" status
- Phone number must be E.164 format


3. OPENAI GPT API (OPTIONAL)
-----------------------------
Model: gpt-4o-mini (cost-effective option)
Pricing: $0.150/1M input tokens, $0.600/1M output tokens
Estimated Cost: ~$0.0005 per person (0.05 cents)

Features:

A) Address Correction
- Fixes malformed addresses that fail API validation
- Expands abbreviations (St→Street, Ave→Avenue)
- Corrects state codes, ZIP formats
- Handles OCR errors and typos
- Automatically retries API call with corrected address
- Max 2 correction attempts per address

B) Contact Filtering & Ranking
- Analyzes all persons from phone + address lookups
- Identifies original customer vs. relatives/associates
- Ranks contacts by likelihood of success
- Provides confidence scores (High/Medium/Low)
- Lists all phone numbers per person with call priority
- Recommends calling strategy and best times
- Calculates acceptance probability

CACHING SYSTEM
================================================================================

DUAL CACHING ARCHITECTURE
--------------------------
The system implements two complementary caching layers:

1. EXCEL CACHE (Output File)
   - Stores processed results in output Excel file
   - Reloads existing data when processing same file
   - Prevents duplicate processing of same phone numbers
   - Scope: Per output file
   - Format: Excel rows with all data

2. PERMANENT JSON CACHE (lead_processor_cache.json)
   - Stores raw API responses across all sessions
   - Persists between application runs
   - Shared across all Excel files
   - Thread-safe with file locking (portalocker)
   - Atomic writes with temp file + rename
   - Stores: phone lookup + address lookup + AI analysis

Cache Key: Normalized phone number (E.164 format)
Example: "(415) 555-1234" → "+14155551234"

Cache Entry Structure:
{
  "timestamp": "2025-01-25T10:30:00Z",
  "reverse_phone": {...},      // Full API response
  "reverse_address": {...},    // Full API response
  "ai_analysis": {...}         // AI filtering results
}

Benefits:
- Reduces API costs (tracks savings in USD)
- Faster processing (cache hits vs. API calls)
- Consistent results across sessions
- Statistics: hit rate, calls saved, estimated savings

DATA PROCESSING PIPELINE
================================================================================

INPUT DATA REQUIREMENTS
-----------------------
Excel file with columns:
- name: Customer name (string)
- phone: Phone number (any format, will be normalized)
- address: Full address or street (string)
- city: City name (optional if in address)
- state: State code (optional if in address)
- zip: ZIP code (optional if in address)
- activity_score: Numeric score (filters <= 30)

PROCESSING FLOW
---------------
1. Load Excel file with pandas
2. Filter rows: activity_score <= 30
3. For each row:
   a. Normalize phone number to E.164 format
   b. Check permanent cache for existing data
   c. If not cached:
      - Perform reverse phone lookup
      - Parse address if needed (smart parser)
      - Perform reverse address lookup
      - If address fails and AI enabled: correct and retry
      - Store results in permanent cache
   d. Extract all persons from API responses
   e. If AI enabled: filter and rank contacts
   f. Create output rows (one per person)
4. Generate formatted Excel output
5. Apply color coding and dropdowns

ADDRESS PARSING
---------------
Smart parser handles various formats:
- "1624 Clementson Drive San Antonio TX 78258 USA"
- "123 Main St, Austin, TX 78701"
- "456 Oak Avenue Apt 2B Houston TX 77001"

Parsing Logic:
1. Remove country code (USA, US, etc.)
2. Extract ZIP code (5 or 9 digits)
3. Extract state (2-letter code or full name)
4. Split remaining into street + city
5. Handle street suffixes (St, Ave, Blvd, etc.)
6. Handle secondary designators (Apt, Suite, Unit, etc.)

State Name Mapping: All 50 US states supported

ERROR HANDLING
================================================================================

API ERROR HANDLING
------------------
- Network timeouts: 10 second timeout on CloudTalk calls
- Connection errors: User-friendly error messages
- Invalid responses: JSON parsing with fallback
- Rate limiting: 0.1 second delay between calls
- Authentication failures: Clear error messages

CACHE ERROR HANDLING
--------------------
- Corrupted cache file: Recreate from scratch
- File locking failures: Graceful degradation
- JSON serialization errors: Skip problematic entries
- Disk space issues: Log errors, continue processing

AI ERROR HANDLING
-----------------
- API key missing: Disable AI features
- Invalid JSON responses: Attempt repair, fallback to standard view
- Token limit exceeded: Truncate input, retry
- Model unavailable: Log error, continue without AI

USER INPUT VALIDATION
---------------------
- Required fields: Check before processing
- File existence: Validate before loading
- Phone format: Normalize automatically
- Excel structure: Validate columns exist
- Starting row: Must be positive integer


THREADING & CONCURRENCY
================================================================================

BACKGROUND THREADS
------------------
1. Bulk Processing Worker
   - Runs in daemon thread
   - Updates GUI via root.after()
   - Handles stop signal gracefully

2. AI Preloading Thread
   - Preloads AI results for next N persons
   - Prevents duplicate requests with queue
   - Thread-safe with locks
   - Daemon thread, auto-cleanup

THREAD SAFETY
--------------
- Cache operations: threading.Lock()
- AI results: threading.Lock()
- Preload queue: threading.Lock()
- File operations: portalocker (cross-platform)
- GUI updates: root.after() for thread safety

SYNCHRONIZATION
---------------
- AI results tracked by person key (phone + name)
- Prevents cross-person contamination
- Validates results match current person
- Clears stale results on navigation
